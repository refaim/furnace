# Wave64 Demux Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent silent loss of Blu-ray LPCM audio tracks when eac3to emits Wave64 (`.w64`) instead of WAV for streams larger than 4 GB. Wave64 files are transcoded to FLAC via eac3to between demux and mux, so `mkvmerge` receives a format it can handle.

**Architecture:** New narrow port `PcmTranscoder` in `furnace/core/ports.py`, implemented by `Eac3toAdapter.transcode_to_flac`. `DiscDemuxer` gains an optional `pcm_transcoder` constructor arg and a private `_transcode_w64_files` method invoked between `port.demux_title()` and the mux decision. Everything downstream (mkvmerge, analyzer, planner, executor) stays untouched — the FLAC track flows through the existing `DECODE_ENCODE → AAC` path.

**Tech Stack:** Python 3.13, `unittest.mock` for subprocess mocking, pytest with coverage enforced at 100 % line + branch via Makefile. TDD mandatory. All lint/typecheck/test invocations go through the Makefile (`make test`, `make check`), never `uv run pytest` directly.

**Design spec:** `docs/superpowers/specs/2026-04-24-w64-demux-fix-design.md`

**Commit policy:** NO intermediate commits. The user commits only when explicitly told to at the end (per `CLAUDE.md` and standing preference). Every task ends with `make test` (or `make check` on final tasks); a single commit step sits at the end of the plan and is executed only on user approval.

```python
    def _transcode_w64_files(self, files: list[Path]) -> list[Path]:
        """Replace any Wave64 (.w64) entries in ``files`` with transcoded FLAC.

        eac3to sometimes writes PCM to Wave64 when the stream exceeds the 4 GB
        WAV limit; mkvmerge cannot read Wave64, so we transcode to FLAC
        (lossless) before muxing. The original .w64 is deleted on success.

        - If no .w64 is present: return ``files`` unchanged.
        - If pcm_transcoder is None and at least one .w64 is present: raise
          RuntimeError (fail fast rather than silently dropping the track).
        - If the transcoder returns non-zero rc for any file: raise
          RuntimeError, leaving the .w64 on disk for inspection.
        """
        if not any(f.suffix.lower() == ".w64" for f in files):
            return files
        if self._pcm_transcoder is None:
            w64_names = [f.name for f in files if f.suffix.lower() == ".w64"]
            msg = (
                "pcm_transcoder not configured; cannot handle Wave64 demux "
                f"output: {w64_names}"
            )
            raise RuntimeError(msg)

        result: list[Path] = []
        for f in files:
            if f.suffix.lower() != ".w64":
                result.append(f)
                continue
            flac_path = f.with_suffix(".flac")
            logger.info("Transcoding Wave64 to FLAC: %s -> %s", f.name, flac_path.name)
            rc = self._pcm_transcoder.transcode_to_flac(f, flac_path)
            if rc != 0:
                msg = f"eac3to transcode of {f.name} to FLAC failed (rc={rc})"
                raise RuntimeError(msg)
            f.unlink()
            result.append(flac_path)
        return result
```

# Furnace — Спецификация проекта

## 1. Обзор

Furnace — инструмент для перекодирования и архивации личной коллекции фильмов на NAS. Принимает на вход видеофайлы (или директории), анализирует их, предлагает интерактивный выбор дорожек и выдаёт оптимизированные MKV-файлы с HEVC-видео, обработанным аудио и правильно размеченными субтитрами.

Работает в два этапа:
1. **Планирование** — интерактивный: пользователь проходит по всем файлам, выбирает дорожки, подтверждает crop. Результат — JSON-файл плана.
2. **Выполнение** — автономный: программа читает план и кодирует всё без вопросов.

Разделение позволяет ответить на все вопросы разом, запустить кодирование и уйти спать.

## 2. Вход и выход

### 2.1 Источники

- Одиночный видеофайл
- Директория (рекурсивный обход)
- Satellite-файлы: если рядом с `movie.mkv` лежат `movie.rus.ac3`, `movie.eng.srt` и т.п., они считаются дополнительными дорожками того же фильма

### 2.2 Выход

- Контейнер MKV (всегда)
- Структура выходной директории зеркалирует структуру источника
- Имена выходных файлов очищаются от запрещённых символов Windows (`< > / : \ | ? *`, двойные кавычки → одинарные, точка в конце убирается)
- Опциональный файл переименования (`--names map.txt`):
  ```
  film.2020.bdrip.x264-group.mkv = Название Фильма (2020)
  ```
  Расширение `.mkv` добавляется автоматически.

### 2.3 Логика пропуска

Файл пропускается, если:
- Выходной файл уже существует, ИЛИ
- В источнике есть MKV-тег `ENCODER`, начинающийся с `Furnace/`

## 3. Интерфейс командной строки

```
furnace plan <source> -o <destination> [options]
furnace run <plan.json>
```

### 3.1 `furnace plan`

Сканирует источник, пробирует все файлы, показывает TUI для выбора дорожек и подтверждения crop. Сохраняет результат в JSON-файл плана.

Опции:
- `-o <path>` — выходная директория (обязательно)
- `--lang <code> [<code> ...]` — языковой фильтр (например `rus eng`). Действует на аудио и субтитры. Если для языка ровно одна дорожка — выбирается автоматом. Если несколько — показывается TUI.
- `--names <path>` — файл переименования
- `--dry-run` — показать что будет запланировано, не сохранять файл плана
- `--vmaf` — включить оценку качества VMAF в план

### 3.2 `furnace run`

Читает файл плана и выполняет все ожидающие задачи. Без интерактивных вопросов. Можно прервать по ESC (грациозная остановка текущего кодирования) и продолжить позже (перезапустить ту же команду).

## 4. Файл плана (JSON)

Содержит список задач (jobs). Каждая задача — один выходной файл:

- Пути к исходным файлам (основное видео + satellite-файлы)
- Путь к выходному файлу
- Параметры кодирования видео (CQ, crop, деинтерлейс, цветовое пространство, HDR-метаданные)
- Выбранные аудиодорожки с инструкциями обработки (копировать / denorm / перекодировать)
- Выбранные дорожки субтитров с инструкциями обработки (копировать / перекодировать в UTF-8)
- Вложения для копирования (шрифты)
- Источник глав
- Статус задачи: `pending` | `done` | `error`
- Сообщение об ошибке (если статус `error`)

При завершении каждой задачи файл плана обновляется на диске — resume работает после прерывания.

## 5. Видео

### 5.1 Кодек

HEVC (H.265) через NVIDIA NVENC (`hevc_nvenc`). Всегда 10-bit (профиль `main10`, формат пикселей `p010le`).

Параметры NVENC (перенесены из Crucible):
- Preset: p5
- Tune: uhq
- Управление битрейтом: VBR с постоянным качеством (CQ)
- Spatial AQ: включён
- Temporal AQ: включён
- RC Lookahead: 32 кадра
- Multipass: qres
- Forced IDR: включён
- GOP: 5-секундный интервал ключевых кадров (рассчитывается из FPS источника)

### 5.2 Качество (CQ)

Адаптивный CQ на основе разрешения источника (площадь в пикселях), интерполируется:

| Разрешение | CQ |
|---|---|
| SD (854x480) | 22 |
| 720p (1280x720) | 24 |
| 1080p (1920x1080) | 25 |
| 1440p (2560x1440) | 28 |
| 4K (3840x2160) | 31 |

Промежуточные разрешения интерполируются линейно по площади пикселей.

### 5.3 Crop

Включён по умолчанию. Используется ffmpeg `cropdetect` для обнаружения чёрных полос. Найденные значения показываются пользователю на этапе планирования для подтверждения (принять / отклонить / изменить). После подтверждения значения выравниваются по сетке 16x8 (требование NVENC).

### 5.4 Деинтерлейс

Автоопределение через ffprobe `field_order`. Если чересстрочное (TFF или BFF), применяется `bwdif_cuda` автоматически без запроса.

### 5.5 Цветовое пространство

Принудительная установка по разрешению:
- HD (>= 720p): BT.709
- SD (< 720p): BT.601
- BT.2020 (HDR-источник): не изменяется

### 5.6 Цветовой диапазон

Всегда Limited (16-235).

### 5.7 HDR

Если источник содержит метаданные HDR10 (mastering display color volume, content light level), они пробрасываются в выход через ffmpeg SEI side data.

### 5.8 Dolby Vision и HDR10+

Если в источнике обнаружен Dolby Vision или HDR10+ (динамические метаданные), файл пропускается с предупреждением. Кодирование не выполняется. Статический HDR10 и HLG поддерживаются полностью (см. 5.7).

### 5.9 Защита от раздувания

После кодирования, если выходной файл больше исходного — выход удаляется, задача помечается как ошибка. Предотвращает бессмысленное перекодирование уже эффективно сжатых источников.

## 6. Аудио

### 6.1 Выбор дорожек

Интерактивный TUI (Textual) на этапе планирования:

```
Аудиодорожки:                   [Пробел] выбор  [Enter] превью  [Вверх/Вниз] навигация

▸ [×]  rus  AC3 5.1    448 kbps   "Дубляж"
  [ ]  rus  AAC 2.0    128 kbps   "Закадровый"
  [×]  eng  TrueHD 7.1            "Original"

  [Готово]
```

- Стрелки вверх/вниз — навигация
- Пробел — вкл/выкл дорожку
- Enter — превью дорожки в mpv (открывает видео с этим аудио)
- Enter на `[Готово]` — подтвердить выбор

С фильтром `--lang`: если для языка ровно одна дорожка — выбирается автоматом. Если несколько — показывается TUI только с дорожками выбранных языков.

Первая выбранная дорожка становится дорожкой по умолчанию в MKV.

### 6.2 Конвейер обработки

| Кодек источника | Действие |
|---|---|
| AAC (любой вариант) | Копировать как есть |
| AC3, EAC3, DTS (core) | eac3to denormalize → копировать |
| DTS-HD MA, DTS-HRA, DTS-ES, TrueHD, FLAC, PCM | eac3to decode в WAV → qaac64 encode в AAC |
| Экзотика (WMA, AMR, Vorbis, MP2, MP3 и т.д.) | ffmpeg decode в WAV → qaac64 encode в AAC |
| Неизвестный кодек | Предупреждение, пропуск файла |

Параметры qaac64: `--tvbr 91 --quality 2 --rate keep --no-delay --threading`

### 6.3 Задержка аудио

Определяется из ffprobe `start_pts`. Если не ноль:
- Для дорожек, проходящих через eac3to: задержка передаётся аргументом `+Xms` / `-Xms`
- Для дорожек, копируемых как есть: задержка передаётся в mkvmerge через `--sync`

### 6.4 Белый список кодеков

Обрабатываются только известные кодеки. Если аудиодорожка содержит кодек не из белого списка, весь файл пропускается с предупреждением. Batch-обработка продолжается со следующего файла.

## 7. Субтитры

### 7.1 Выбор дорожек

Тот же TUI что и для аудио. Enter открывает превью субтитров в mpv поверх видео.

### 7.2 Обработка

| Кодек источника | Действие |
|---|---|
| SRT | Копировать, перекодировать в UTF-8 при необходимости |
| ASS/SSA | Копировать, перекодировать в UTF-8 при необходимости |
| PGS | Копировать как есть |
| VOBSUB | Копировать как есть |
| Неизвестный кодек | Предупреждение, пропуск файла |

Определение кодировки текстовых субтитров через `charset-normalizer`. Если файл не валидный UTF-8, он перекодируется из определённой исходной кодировки в UTF-8.

### 7.3 Детекция форсированных субтитров

Трёхэтапный алгоритм (из mvtools):

1. **Ключевые слова в имени файла**: `forced`, `форсир`, `только надписи` (исключая `normal`) → отметить как forced
2. **Ключевые слова в имени дорожки**: `forced`, `caption` → отметить как forced. `sdh` → отметить как полные (не forced)
3. **Статистика**: для каждого языка, если дорожка субтитров содержит < 50% количества кадров или строк от самой большой дорожки на том же языке → отметить как forced

### 7.4 Вложения

Все вложения из исходного файла (шрифты, изображения) копируются в выходной MKV. Необходимо для корректного отображения стилизованных ASS-субтитров.

### 7.5 Белый список кодеков

Аналогично аудио: неизвестные кодеки субтитров приводят к пропуску файла с предупреждением.

## 8. Контейнер и метаданные

### 8.1 Сборка

Финальная сборка через `mkvmerge`. Все обработанные дорожки (видео, аудио, субтитры), главы и вложения собираются в один MKV-файл.

### 8.2 mkclean

После сборки запускается `mkclean` для переноса индекса поиска в начало файла. Необходимо для плавного воспроизведения по сети (SMB) без задержек при перемотке.

### 8.3 Метаданные

- Тег `ENCODER` установлен в `Furnace/X.X.X` (версия инструмента)
- Все названия дорожек удаляются
- Язык видеодорожки — `und` (не определён)
- Языки аудио- и дорожек субтитров сохраняются из источника
- Все прочие теги удаляются (`--no-track-tags --no-global-tags --disable-track-statistics-tags`)

### 8.4 Главы

Копируются из источника при наличии.

### 8.5 Дорожки по умолчанию

- Первая выбранная аудиодорожка отмечается как default
- Первая выбранная дорожка субтитров отмечается как default (если forced — также отмечается как forced)

## 9. Пакетная обработка

### 9.1 Сканирование директории

Рекурсивный обход исходной директории. Для каждого найденного видеофайла:
1. Обнаружить satellite-файлы (то же имя, другое расширение)
2. Пробировать метаданные
3. Проверить логику пропуска (уже обработан или выход существует)
4. Показать в TUI для выбора дорожек

### 9.2 Двухфазное выполнение

- **Фаза 1 (plan)**: все интерактивные решения принимаются здесь. Результат: JSON-файл плана.
- **Фаза 2 (run)**: читает план, кодирует последовательно. Без вопросов. Обновляет статус задачи в JSON после каждого файла.

### 9.3 Возобновление

`furnace run plan.json` пропускает задачи со статусом `done`, повторяет задачи со статусом `error` или `pending`. Безопасно перезапускать после прерывания.

### 9.4 VMAF

Опционально, включается флагом `--vmaf` при планировании. Если включено, после каждого кодирования рассчитывается оценка VMAF (фильтр libvmaf в ffmpeg с ускорением CUDA) и сохраняется в файле плана.

### 9.5 Итоговый отчёт

После завершения всех задач выводится сводка:
- Файлов обработано / пропущено / с ошибками
- Общий размер до и после
- Общая экономия размера
- Средняя оценка VMAF (если включено)

### 9.6 Грациозное прерывание

Клавиша ESC во время кодирования:
- Завершает дерево процессов ffmpeg (через psutil)
- Статус текущей задачи остаётся `pending`
- Программа завершается
- Возобновление: `furnace run plan.json`

## 10. Пользовательский интерфейс

### 10.1 Фаза планирования (Textual TUI)

- Виджеты выбора дорожек с клавиатурной навигацией
- Диалог подтверждения crop
- Отображение информации о файле (разрешение, кодеки, длительность, размер)

### 10.2 Фаза кодирования (Rich)

- Прогресс-бар для каждого файла с fps / скорость / ETA
- Информация об исходном и выходном файле
- Последние N сообщений лога ffmpeg
- Общий прогресс batch (X из Y файлов)

### 10.3 Интеграция с mpv

На этапе планирования Enter на дорожке открывает mpv для превью:
- Аудиодорожка: воспроизводит видео с выбранным аудио
- Дорожка субтитров: воспроизводит видео с выбранными субтитрами поверх
- Пользователь закрывает mpv — возврат в TUI

## 11. Архитектура

### 11.1 Паттерн: Ports & Adapters (гексагональная архитектура)

Ядро (чистая логика) не зависит от внешних инструментов, UI и файловой системы. Связь через протоколы (абстракции).

```
furnace/
├── __main__.py              # точка входа
├── cli.py                   # typer CLI (plan, run)
│
├── core/                    # ядро — чистый Python, без I/O
│   ├── models.py            # Movie, Track, Job, Step (dataclasses)
│   ├── rules.py             # белые списки кодеков, маршрутизация аудио
│   ├── quality.py           # интерполяция CQ, crop alignment
│   ├── detect.py            # forced subs, interlace, HDR/DV detection rules
│   └── ports.py             # протоколы (Prober, Encoder, Muxer и т.д.)
│
├── services/                # оркестрация — соединяет ядро с адаптерами
│   ├── scanner.py           # обход директорий, группировка satellite files
│   ├── analyzer.py          # пробирование файлов, заполнение моделей
│   ├── planner.py           # применяет rules к моделям → список Job
│   └── executor.py          # читает план, выполняет Job последовательно
│
├── adapters/                # реализации портов — обёртки над инструментами
│   ├── ffmpeg.py            # encode, probe, vmaf, cropdetect
│   ├── eac3to.py            # denorm, decode lossless
│   ├── qaac.py              # encode AAC
│   ├── mkvmerge.py          # mux, metadata
│   ├── mkclean.py           # оптимизация индекса
│   └── mpv.py               # превью дорожек
│
├── ui/                      # всё что видит пользователь
│   ├── tui.py               # Textual — выбор дорожек, crop confirm
│   └── progress.py          # Rich — прогресс кодирования
│
└── plan.py                  # сериализация/десериализация JSON плана
```

### 11.2 Направление зависимостей

```
UI ──→ Services ──→ Core ←── Adapters
                      ↑
                   ports.py (протоколы)
```

- Core не импортирует ничего из остальных слоёв
- Adapters реализуют протоколы из Core (инверсия зависимости)
- Services принимают адаптеры через конструктор/аргументы
- UI вызывает Services

### 11.3 Качество кода

**Линтер:** ruff (строгий режим)

```toml
[tool.ruff.lint]
select = ["ALL"]
ignore = ["D", "ANN101", "ANN102"]  # docstrings не требуем, self/cls не аннотируем
```

**Типизация:** mypy (strict)

```toml
[tool.mypy]
strict = true
warn_return_any = true
warn_unused_configs = true
disallow_untyped_defs = true
disallow_any_generics = true
```

**Тесты:** pytest

Покрытие тестами:
- `core/` — юнит-тесты (чистые функции, без моков)
- `adapters/` — интеграционные тесты (на реальных файлах, опционально)
- `services/` — тесты с моками адаптеров через протоколы

**Pre-commit hooks:**
- ruff check + ruff format
- mypy
- pytest (быстрые юнит-тесты)

### 11.4 Референсы

- **Crucible** (`C:\Users\Roma\Dev\crucible\crucible.py`) — референс по работе с ffmpeg/ffprobe, NVENC-параметрам, VMAF, прогресс-бару, грациозному завершению
- **mvtools** (`C:\Users\Roma\Dev\mvtools`) — референс по общей логике программы: сканирование файлов, satellite files, выбор дорожек, аудио-конвейер (eac3to/qaac), субтитры (forced detection, кодировки), mkvmerge-сборка, метаданные

## 12. Вне скоупа

- Масштабирование / даунскейл видео
- Даунмикс аудио
- Профили устройств
- Переименование сериалов
- CPU-кодирование (libx264 / libx265)
- Кодек AV1
- Remux с DVD / Blu-ray дисков (возможно в будущем)
- Удаление оригиналов
- Принудительная установка DAR
- Генерация batch-скриптов
